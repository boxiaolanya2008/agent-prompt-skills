# Diff优化技能

## 问题背景

Agent-IDE在代码修改时存在严重问题：无法进行精确的差异化比较(diff)，导致整文件重写而非仅修改变更部分。这严重影响了代码修改效率和用户体验。

## 核心目标

- **精确差异识别**: 准确识别原始代码与修改后代码的差异
- **片段化输出**: 仅输出变更的代码片段，避免整文件重写
- **结构保留**: 保持代码的原始结构和格式
- **多语言支持**: 支持主流编程语言的语法解析

## 核心功能模块

### 1. 代码差异分析引擎

#### 精确差异识别
- **行级对比**: 逐行对比原始代码和修改后代码
- **字符级分析**: 对变更行进行字符级的精确分析
- **语义理解**: 理解代码语义，识别真正的变更而非格式差异
- **上下文关联**: 分析变更的上下文，确保修改的完整性

#### 变更类型识别
- **新增识别**: 准确识别新增的代码行和代码块
- **删除识别**: 准确识别删除的代码行和代码块
- **修改识别**: 准确识别修改的代码行和代码块
- **移动识别**: 识别代码位置的移动和调整

### 2. 代码片段提取系统

#### 智能片段提取
- **最小化原则**: 提取最小的变更代码片段
- **上下文保留**: 保留必要的上下文信息确保代码可理解
- **边界识别**: 智能识别代码块的边界，避免截断
- **完整性保证**: 确保提取的代码片段完整且可执行

#### 格式保持
- **缩进保留**: 严格保留原始代码的缩进格式
- **空行处理**: 正确处理空行和注释行
- **语法高亮**: 保持代码的语法结构和可读性
- **编码一致**: 确保编码格式的一致性

### 3. 多语言语法解析器

#### 主流语言支持
- **JavaScript/TypeScript**: 完整支持ES6+语法和TypeScript特性
- **Python**: 支持Python 3.x的所有语法特性
- **Java**: 支持Java 8+的语法特性
- **C/C++**: 支持C11/C++17标准
- **Go**: 支持Go语言的语法特性
- **Rust**: 支持Rust语言的语法特性
- **其他语言**: 支持主流编程语言的语法解析

#### 语法感知
- **AST解析**: 使用抽象语法树进行深度语法分析
- **作用域识别**: 识别变量和函数的作用域
- **依赖分析**: 分析代码间的依赖关系
- **语法验证**: 验证修改后代码的语法正确性

### 4. Diff输出格式化器

#### 标准Diff格式
```
--- 原始文件
+++ 修改文件
@@ 行号,行数 行号,行数 @@
 上下文代码
-删除的代码
+新增的代码
 上下文代码
```

#### 增强Diff格式
- **颜色标记**: 使用颜色区分新增、删除和修改
- **行号标注**: 清晰标注变更的行号范围
- **上下文显示**: 显示变更前后的上下文代码
- **统计信息**: 提供变更的统计信息（新增行数、删除行数等）

## 使用示例

### 示例1：函数修改
**原始代码**:
```python
def calculate_sum(a, b):
    return a + b
```

**修改后代码**:
```python
def calculate_sum(a, b, c=0):
    return a + b + c
```

**Diff输出**:
```
--- original.py
+++ modified.py
@@ -1,2 +1,2 @@
-def calculate_sum(a, b):
-    return a + b
+def calculate_sum(a, b, c=0):
+    return a + b + c
```

### 示例2：代码块新增
**原始代码**:
```javascript
function validate(input) {
    return input !== null;
}
```

**修改后代码**:
```javascript
function validate(input) {
    if (input === null || input === undefined) {
        return false;
    }
    return input !== null;
}
```

**Diff输出**:
```
--- original.js
+++ modified.js
@@ -1,3 +1,6 @@
 function validate(input) {
+    if (input === null || input === undefined) {
+        return false;
+    }
     return input !== null;
 }
```
