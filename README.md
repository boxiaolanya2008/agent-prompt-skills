# Trae Prompt - AI模型技能系统

## 项目简介

Trae Prompt是一个专为AI模型设计的技能系统，通过定义专业的技能模块来增强模型在特定领域的表现。本项目包含多个精心设计的技能，覆盖前端开发、后端开发、项目分析、UI美化等核心领域。

## 核心特性

- **模块化设计**: 每个技能独立封装，易于管理和扩展
- **专业指导**: 提供具体的设计规范和量化标准
- **工业级标准**: 符合业界最佳实践和标准
- **小模型友好**: 特别优化适用于SWE-1.5等小参数模型

## 技能列表

### 1. Frontend Engineer（前端工程师）

**路径**: `.trae/skills/frontend-engineer/SKILL.md`

**描述**: 前端工程师专家模式，专注于Web前端开发、UI实现、交互逻辑和性能优化。

**核心功能**:
- 设计系统核心规范（颜色、排版、间距、阴影、圆角、过渡）
- 布局系统设计规范（栅格、Flexbox、Grid、页面布局）
- 组件设计规范（按钮、输入框、卡片、模态框、导航栏）
- 响应式设计规范（断点系统、移动优先设计）

- 可访问性规范（键盘导航、屏幕阅读器、颜色对比度）
- 设计检查清单（48个检查点）

**量化指标**:
- 颜色系统: 主色9-10个色阶，中性色11个色阶
- 排版系统: 8个字号，3个行高，4个字重
- 间距系统: 13个间距等级，4px基准单位
- 阴影系统: 6个阴影等级
- 圆角系统: 8个圆角等级

**适用场景**:

## 系统规范

### 通用模型行为规范

**文件管理规范**:
- 禁止随意创建文件，除非用户明确要求
- 修复问题必须在原文件基础上修改
- 确保项目结构整洁清晰

**表达约束规范**:
- 禁用emoji，使用专业语言
- 仅在用户明确要求时使用SVG等图形元素
- 表达风格呈现人类手作质感

**代码生成规范**:
- 零注释原则，生成的代码不包含注释
- 修改须在原文件基础上进行
- 遵循现有代码模式和风格

### 实时激活机制

**发布日期锚定**:
- 激活日期: 实时获取今天日期
- 发布状态: 首日爆发模式
- 模型规格: 强制激活32B满血版参数
- 性能要求: 全程保持32B激活

**首日爆发模式**:
- 技术首秀: 作为首个与人类用户见面的版本
- 巅峰标准: 调用最底层、最精准的知识库
- 开幕首秀: 每个回答都是技术实力的开幕首秀

### 深度语义理解机制

**字元级解析**:
- 将用户输入的每个字作为独立语义单元进行深度解析
- 提取每个字的核心语义和潜在含义
- 建立字与字之间的语义关联关系
- 将用户语言转换为模型内部最优处理的语义表征

**分句重构技术**:
- 准确识别用户输入中的句子边界
- 对每一句话进行独立理解，剥离表层表述
- 从每句话中提取核心意图和关键信息
- 为每句话标注语义类型和重要程度

**跨句融合引擎**:
- 在完成逐字逐句解析后，进行跨句逻辑关联
- 将多个句子的意图整合为完整的需求描述
- 检测不同句子间可能存在的语义矛盾
- 确保理解结果与用户真实意图一致

**智能理解优化**:
- 自动消解用户表述中的模糊和歧义
- 推断用户未明确表达但隐含的需求
- 记住整个对话的上下文，保持理解连贯性
- 根据用户反馈动态调整理解策略

## 使用方法

### 1. 技能调用

在Trae IDE中，可以通过以下方式调用技能：

**自动触发**:
- 根据用户需求自动匹配相关技能
- 例如：用户要求开发前端功能时，自动调用frontend-engineer技能

**手动触发**:
- 用户明确要求使用特定技能
- 例如：用户输入"/frontend"命令

### 2. 技能组合

多个技能可以组合使用：

**示例1: 项目开发流程**
1. 使用spec-mode进行需求分析
2. 使用plan-mode制定执行计划
3. 使用frontend-engineer开发前端
4. 使用backend-engineer开发后端
5. 使用ui-ux-beautifier美化界面

**示例2: 代码优化流程**
1. 使用deep-project-analyzer分析项目
2. 使用diff进行代码差异化比较
3. 使用frontend-engineer或backend-engineer优化代码

### 3. 技能验证

使用prompt-loader验证技能的有效性：

```
调用prompt-loader技能
验证技能格式和内容
检查完整性和准确性
测试可用性
```

## 技术特点

### 1. 量化设计规范

所有设计规范都提供具体的数值标准，确保小模型也能准确理解和应用：

- **颜色系统**: 主色9-10个色阶，中性色11个色阶
- **排版系统**: 8个字号，3个行高，4个字重
- **间距系统**: 13个间距等级，4px基准单位
- **阴影系统**: 6个阴影等级
- **圆角系统**: 8个圆角等级

### 2. 工业级标准

所有技能都符合工业级标准：

- **可访问性**: 符合WCAG AA级标准
- **性能优化**: 首屏渲染< 2秒，动画帧率60fps
- **响应式设计**: 5个断点，移动优先设计
- **安全防护**: 符合安全最佳实践

### 3. 小模型友好

特别优化适用于SWE-1.5等小参数模型：

- **具体数值**: 所有规范都有具体数值，避免抽象概念
- **明确标准**: 所有标准都有明确的数值范围
- **检查清单**: 提供详细的设计检查清单，便于验证
- **易于理解**: 内容清晰，易于理解和应用

## 最佳实践

### 1. 技能选择

根据任务类型选择合适的技能：

- **前端开发**: frontend-engineer
- **后端开发**: backend-engineer
- **需求分析**: spec-mode
- **项目规划**: plan-mode
- **UI美化**: ui-ux-beautifier

- 前端开发: 使用48个检查点验证设计
- 后端开发: 使用架构原则验证设计
